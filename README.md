# 🐾 保護猫カフェ「SakuRa Cafe」Webサイト（架空プロジェクト）

動物好きな20〜40代の女性をターゲットにした、架空の保護猫カフェ「SakuRa Cafe」のプロモーション・Webサイトです。
当初は静的なHTML/CSSで構築し、その後に開発・運用の効率化（DRY原則）を見据えて、静的サイトジェネレーター「Jekyll」を用いてコンポーネント化・再構築を行いました。

## 🎨 デザインコンセプト & UI/UXのこだわり

### 1. ターゲットに寄り添ったカラー設計
20〜40代の女性が「優しい癒しの時間」をイメージできるよう、店名である桜を想起させる**サーモンピンク**、大人っぽい落ち着きを与える**ブルーグリーン**、全体を柔らかく包む**ベージュ**の3色をベースに配色しました。

### 2. ユーザーの不安を解消する情報設計
「猫カフェに興味はあるが、衛生面が気になる」という潜在的な心理ハードルを下げるため、**「猫広場とティールームが完全に分かれている」という店舗最大の強み**をファーストビュー以降で明確に視覚化しました。

### 3. 社会的意義のストーリー化
「カフェの利用自体が保護活動の支援になる」という仕組みや、猫スタッフがすべて保護猫であるというストーリーを丁寧に配置し、共感から来店へと繋げる導線を意識しました。落ち着いたトーンを維持しつつ、随所に愛らしい猫の写真を配置して直感的な魅力を伝えています。

- **Frontend:** HTML5 / CSS3（SCSS）
- **Dev Tools:** Git / GitHub, VS Code

### 💻 コーディング・運用のこだわり
- **Jekyllによるデータ管理の効率化:**
  メニュー、料金プラン、店舗情報など「今後変動する可能性が高いデータ」は、HTMLに直書きせず `_data/` 内のYAMLファイルで一元管理しています。これにより、将来の店舗展開やメニュー変更時にもHTMLを汚さず、安全かつ瞬時に更新できる運用コストの低い設計にしています。
- **SCSSを用いたコンポーネント管理:**
  `_sass/` ディレクトリ内でスタイルを機能・パーツごとに細かくモジュール化（分割管理）しています。変数（Variables）を活用してカラーパレットを一括管理しているため、ブランドカラーの微調整などにも柔軟に対応できる破綻しにくいCSS設計を意識しました。
- **拡張性を意識した設計:**
  将来的な機能拡張を見据え、データ構造を疎結合に保っています。

## 📂 ディレクトリ構造
```text
├── _data/              # navigationなどのデータをYAMLで管理
├── _includes/          # ヘッダー、フッター、各セクションをモジュール化
├── _layouts/           # 共通レイアウト
├── _sass/              # スタイルを機能ごとに分割管理（SCSS）
└── assets/
    ├── css/style.css   # _sassからインポート・ビルドされるメインCSS
    └── js/main.js      # スクロールやスライダーアニメーションを制御するバニラJS
```
